PAGX AI Skills

Give your AI coding assistant the ability to create and optimize PAGX vector graphics.

PAGX AI Skills is a reference documentation package that enables AI coding assistants to generate well-structured PAGX files from text descriptions, optimize existing files for size and rendering performance, and look up PAGX syntax, attributes, and CLI commands.

Under the hood, vector generation is essentially an AI coding process — the model writes structured markup rather than sampling pixels. This means the quality of output improves naturally as AI coding models advance. Over time, less detailed prompting and fewer manual iteration rounds will be needed to achieve the same results.

PAGX was originally designed to describe motion graphics — animations built on top of vector graphics. The format is under active development; the current focus is on refining the static vector graphics foundation. Once animation capabilities are added, AI will be able to go further and directly manipulate these vector elements to produce complex motion effects as well.

The package includes:

  • Format specification essentials and design patterns
  • Step-by-step generation and optimization guides
  • Complete attribute reference with defaults and enumerations
  • CLI tool reference for rendering, validation, and more
  • Practical examples covering icons, UI components, cards, and text

Installation Guide

1. Install Node.js

The Skills package includes a companion command-line tool (pagx-cli) that requires a Node.js runtime. If you don't have Node.js installed yet, follow these steps:

Step 1 — Visit nodejs.org and download the LTS (Long Term Support) version for your operating system.

Step 2 — Run the installer and follow the on-screen instructions. Accept all default settings.

Step 3 — Verify the installation. Open a terminal (Terminal on macOS, Command Prompt or PowerShell on Windows) and run:

node --version

You should see a version number like v22.x.x. Version 18 or later is required.

2. Install CodeBuddy Code

CodeBuddy Code is an AI coding assistant that runs in your terminal. If you haven't installed it yet:

Step 1 — Open a terminal and run the installation command:

npm install -g @anthropic-ai/codebuddy-code

Step 2 — After installation, launch it by typing:

codebuddy

Follow the on-screen prompts to complete the initial setup and authentication.

For detailed installation instructions and system requirements, see the CodeBuddy Code documentation.

3. Install PAGX Skills

After downloading the Skills package above, extract and install it into your project:

Step 1 — Open a terminal and navigate to your project folder (the folder where you want to create PAGX files):

cd /path/to/your/project

Step 2 — Create the Skills directory:

mkdir -p .codebuddy/skills

Step 3 — Extract the downloaded pagx.zip into the Skills directory:

unzip pagx.zip -d .codebuddy/skills/pagx

After extraction, the directory structure should look like:

your-project/
  .codebuddy/
    skills/
      pagx/
        SKILL.md
        references/
          ...

Step 4 — Launch CodeBuddy Code in your project folder:

codebuddy

The PAGX Skills will be automatically detected and loaded.

The pagx-cli command-line tool will be automatically installed when the AI assistant needs it for the first time. This requires the Node.js environment set up in Step 1.

PAGX Skills has been most thoroughly tested with CodeBuddy Code. For best results, select the Claude Opus model in your CodeBuddy Code settings — it currently produces the highest quality output with this Skills package.

Using with Other AI Coding Tools

The guide above uses CodeBuddy Code as an example, but PAGX Skills is a set of structured reference documents following an open convention — it is not tied to any specific tool. If you use a different AI coding assistant that supports a Skills (or similar) mechanism, you only need to:

  1. Ensure Node.js is installed on your system (see Step 1 above) — this is required for the pagx-cli tool that Skills relies on.
  2. Copy the Skills files into the directory designated by your AI tool. Refer to Step 3 above for the extraction process — just replace the .codebuddy/skills/ path with the corresponding directory for your tool.

Getting Started

Once the Skills are installed, you can interact with the AI using natural language. Here are some common operations:

Creating PAGX Files

  • "Create a PAGX file for a blue rounded rectangle button with white text 'Submit'"
  • "Design a card component with an avatar, title, and subtitle"
  • "Make a settings icon using simple geometric shapes, 24x24"
  • "Create a gradient banner with the text 'Welcome Back' in bold"

Modifying and Iterating

  • "Change the button color to green and increase the corner radius"
  • "Make the title text larger and align it to the left"
  • "Add a subtle drop shadow to the card"
  • "The spacing between the icon and the label feels too tight, increase it"

Rendering and Inspection

  • "Render this PAGX file as a screenshot so I can preview it"
  • "Render it at 2x scale for Retina display"
  • "Validate this PAGX file and fix any errors"
  • "Optimize this PAGX file to reduce its size"

Why Vector Generation Matters

Bitmap-based image generation (such as Stable Diffusion or DALL-E) works by sampling from a probability distribution — essentially drawing a lottery ticket each time. The result is never fully controllable: you might generate dozens of images, each with minor imperfections that prevent production use. More critically, there is no way to precisely modify just one part of a bitmap output. Need the button 2 pixels wider? The shadow slightly softer? You have to regenerate the entire image and hope for the best. This inherent unpredictability limits bitmap generation largely to early-stage creative exploration.

Vector generation solves this fundamental problem. Because PAGX is a structured format with precise, editable values, every element can be modified exactly. If something isn't right, you simply describe the change and the AI adjusts that specific part — colors, sizes, positions, spacing — without affecting anything else. With enough iterations, the output can reliably reach production quality.

Tips for Best Results

Iterate to perfection. Don't expect a perfect result on the first try. Instead, treat the process as a conversation: describe what you want, preview the rendered output, then tell the AI what to adjust. Each round of feedback produces precise modifications — not probabilistic re-rolls — so you steadily converge on the ideal result. This multi-round iteration can be fully automated by the AI, or you can step in at any point to provide key aesthetic judgment and structural guidance. This ability to make exact, language-driven modifications across multiple iterations is the core value of this workflow.

Create, don't trace. If you provide a bitmap image as reference, the AI will use it as inspiration for a fresh vector creation — not as a template to reproduce pixel by pixel. Precise bitmap reproduction is not a strength of any current AI model: a bitmap is fundamentally a probabilistic "impression," while a vector graphic is a precise structural description with an exact solution. Deriving exact structure from a probability distribution goes against the nature of how large language models work. Use reference images to convey style and concept, then let the AI create a clean vector interpretation.

Command-Line Tools

The Skills package works with pagx-cli, a companion command-line tool distributed via npm. When used through an AI assistant, the CLI is invoked automatically. You can also install and use it directly: 

npm install -g @libpag/pagx
CommandDescription
pagx renderRender a PAGX file to PNG, WebP, or JPG
pagx validateValidate a PAGX file against the specification
pagx optimizeValidate, optimize structure, and format in one step
pagx formatPretty-print with consistent indentation
pagx boundsQuery precise rendered bounds of layers
pagx font infoQuery font identity and metrics
pagx font embedEmbed fonts into a PAGX file
