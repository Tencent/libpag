简介

PAGX AI Skills 是面向 AI 编程助手的专业技能扩展。安装后，助手可以根据文字描述生成结构化的 PAGX 矢量图形，优化现有文件的体积与渲染性能，并在需要时查询 PAGX 语法、属性和命令行工具用法。

先看传统路径。当前主流的 AI 出图工具（如 Stable Diffusion、DALL-E）主要生成位图，本质上是从概率分布中采样结果。若某个细节不理想，通常只能整张重试，很难只改动单一局部。

PAGX 的路径不同：AI 输出的是结构化标记而不是像素采样。文件中的元素都带有明确、可编辑的参数，因此你可以用自然语言指定局部改动，例如颜色、尺寸、位置和间距，并且不影响其他部分。这种“可精确二次编辑”的能力，是矢量工作流的核心优势。

在这个流程中，你可以选择全自动迭代，也可以在关键节点加入人工审美和结构判断。通过多轮“小步修改 + 渲染校验”，结果会稳定收敛。随着模型能力提升，实现同等效果所需的提示也会逐步减少。

PAGX 最初为动效描述而设计，动效能力建立在矢量静态图基础上。当前阶段重点仍是夯实静态能力；后续动效模块完善后，AI 可在同一套元素结构上继续生成更复杂的动画效果。

安装指南

1. 安装 Node.js

Skills 安装包包含命令行工具 pagx-cli，运行依赖 Node.js。若本机尚未安装，请先完成以下步骤：

第 1 步 — 访问 nodejs.org，下载适用于你操作系统的 LTS（长期支持）版本。

第 2 步 — 运行安装程序，按照屏幕提示操作，保持所有默认设置即可。

第 3 步 — 验证安装是否成功。打开终端（macOS 上的"终端"应用，Windows 上的"命令提示符"或"PowerShell"），输入以下命令：

node --version

正常情况下会显示类似 v22.x.x 的版本号。版本需要 18 或更高。

2. 安装 CodeBuddy Code

CodeBuddy Code 是运行在终端中的 AI 编程助手。若尚未安装，请按以下步骤操作：

第 1 步 — 打开终端，运行以下安装命令：

npm install -g @tencent-ai/codebuddy-code

第 2 步 — 安装完成后，输入以下命令启动：

codebuddy

按照屏幕提示完成初始设置和账号认证。

如需了解详细的安装说明和系统要求，请参阅 CodeBuddy Code 官方文档。

3. 安装 PAGX Skills

下载上方的 Skills 安装包后，将 pagx.zip 解压，把得到的 pagx 文件夹放到项目中的以下位置：

your-project/
  .codebuddy/
    skills/
      pagx/
        SKILL.md
        references/
          ...

如果项目中还没有 .codebuddy/skills/ 文件夹，手动创建即可。然后在项目目录下启动 CodeBuddy Code，PAGX Skills 会被自动识别并加载。

pagx-cli 命令行工具会在 AI 助手首次需要时自动安装，这依赖于第 1 步中搭建的 Node.js 环境。

目前 PAGX Skills 在 CodeBuddy Code 上经过了最全面的测试。为获得最佳效果，建议在 CodeBuddy Code 设置中选择 Claude Opus 模型，该模型在使用此 Skills 时生成质量最高。

在其他 AI 编程工具中使用

上面的步骤以 CodeBuddy Code 为例。PAGX Skills 本身遵循开放约定，不绑定特定工具。若你使用其他支持 Skills（或类似机制）的 AI 助手，通常只需完成以下两件事：

  1. 确保系统已安装 Node.js（参见上方第 1 步）— 这是 Skills 依赖的 pagx-cli 工具运行所必需的。
  2. 将 Skills 文件拷贝到对应目录。参考上方第 3 步的解压过程，只需将 .codebuddy/skills/ 替换为你所用工具指定的 Skills 存放路径即可。

开始使用

安装完成后，可直接用自然语言驱动 AI。下面按“创建→迭代→验证”给出常见示例：

创建 PAGX 文件

  • "创建一个蓝色圆角矩形按钮的 PAGX 文件，按钮上有白色文字 'Submit'"
  • "设计一个卡片组件，包含头像、标题和副标题"
  • "用简单的几何图形做一个设置图标，24x24 尺寸"
  • "创建一个渐变色横幅，上面有加粗文字 'Welcome Back'"

修改与迭代

  • "把按钮颜色改成绿色，圆角再大一些"
  • "标题文字调大一些，改成左对齐"
  • "给卡片加一个淡淡的阴影效果"
  • "图标和文字之间的间距感觉太紧了，拉开一些"

渲染与检查

  • "把这个 PAGX 文件渲染成截图，我看一下效果"
  • "渲染成 2 倍大小，方便在 Retina 屏幕上预览"
  • "校验一下这个 PAGX 文件有没有错误，有的话帮我修复"
  • "优化一下这个 PAGX 文件的体积"

在线预览

你也可以通过 PAGX Playground 在线预览 PAGX 文件 — 只需将任意 .pagx 文件拖放到页面上，即可在浏览器中查看渲染效果。

使用建议

先定目标，再做小步迭代。先描述目标效果，再看渲染结果，最后只反馈需要改动的局部。由于每轮都是可定位、可验证的精确修改（而不是概率重试），结果会更快收敛，也更容易稳定复用。

参考位图，用于“定风格”而非“做临摹”。位图适合传达氛围与构图，不适合逐像素复刻。更有效的做法是让 AI 依据参考图重建为结构清晰的矢量表达，再通过局部参数持续打磨到可用版本。
