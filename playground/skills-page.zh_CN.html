简介

PAGX AI Skills 是一套参考文档包，安装后可以让 AI 编程助手根据文字描述生成结构规范的 PAGX 文件， 优化已有文件的体积和渲染性能，以及查询 PAGX 语法、属性和命令行工具用法。

矢量图生成的本质是一个 AI 编程过程 — 模型编写的是结构化标记语言，而非采样像素。 这意味着输出质量会随着 AI 编程大模型能力的持续提升而自然提高，达到同样效果所需的 提示详细程度和人工迭代次数都会逐渐降低。与位图生成（如 Stable Diffusion、DALL-E） 从概率分布中抽样、无法精确修改输出不同，PAGX 产出的是结构化文件，每个元素都有精确 可编辑的值。哪里不对，只需描述一下改动，AI 就会精确调整那个具体的部分 — 颜色、尺寸、位置、间距 — 而不影响其他任何东西。只要迭代次数足够， 输出就几乎一定可以达到上线标准。

PAGX 格式最初就是为描述动效而设计的 — 动效是在矢量静态图基础上的扩展。 该格式目前仍在持续迭代中，当前阶段的重点是打磨矢量静态图的基础能力。 待动效能力补充完善后，AI 还可以进一步直接操作这些矢量元素来生成复杂的动效。

文档包包含：

  • 格式规范要点与设计模式
  • 分步骤的生成指南和优化指南
  • 完整的属性参考（含默认值和枚举值）
  • 命令行工具参考（渲染、校验等）
  • 实用示例（图标、UI 组件、卡片、文本等）

安装指南

1. 安装 Node.js

Skills 安装包内含配套的命令行工具（pagx-cli），运行时需要 Node.js 环境。 如果你的电脑上还没有安装 Node.js，请按以下步骤操作：

第 1 步 — 访问 nodejs.org，下载适用于你操作系统的 LTS（长期支持）版本。

第 2 步 — 运行安装程序，按照屏幕提示操作，保持所有默认设置即可。

第 3 步 — 验证安装是否成功。打开终端（macOS 上的"终端"应用，Windows 上的"命令提示符"或"PowerShell"），输入以下命令：

node --version

正常情况下会显示类似 v22.x.x 的版本号。版本需要 18 或更高。

2. 安装 CodeBuddy Code

CodeBuddy Code 是一款在终端中运行的 AI 编程助手。如果你还没有安装过，请按以下步骤操作：

第 1 步 — 打开终端，运行以下安装命令：

npm install -g @tencent-ai/codebuddy-code

也可以使用其他包管理器安装：

# Homebrew（macOS / Linux）
brew install Tencent-CodeBuddy/tap/codebuddy-code

# 或无需 Node.js 安装（macOS / Linux）
curl -fsSL https://copilot.tencent.com/cli/install.sh | bash

第 2 步 — 安装完成后，输入以下命令启动：

codebuddy

按照屏幕提示完成初始设置和账号认证。

如需了解详细的安装说明和系统要求，请参阅 CodeBuddy Code 官方文档。

3. 安装 PAGX Skills

下载上方的 Skills 安装包后，将其解压并安装到你的项目中：

第 1 步 — 打开终端，进入你的项目文件夹（即你想要创建 PAGX 文件的目录）：

cd /path/to/your/project

第 2 步 — 创建 Skills 目录：

mkdir -p .codebuddy/skills

第 3 步 — 将下载的 pagx.zip 解压到 Skills 目录中：

unzip pagx.zip -d .codebuddy/skills/pagx

解压后，目录结构应如下所示：

your-project/
  .codebuddy/
    skills/
      pagx/
        SKILL.md
        references/
          ...

第 4 步 — 在项目文件夹中启动 CodeBuddy Code：

codebuddy

PAGX Skills 会被自动识别并加载。

pagx-cli 命令行工具会在 AI 助手首次需要时自动安装，这依赖于第 1 步中搭建的 Node.js 环境。

目前 PAGX Skills 在 CodeBuddy Code 上经过了最全面的测试。为获得最佳效果，建议在 CodeBuddy Code 设置中选择 Claude Opus 模型，该模型在使用此 Skills 时生成质量最高。

在其他 AI 编程工具中使用

以上指南以 CodeBuddy Code 为例，但 PAGX Skills 是一套遵循开放规范的结构化参考文档，不绑定任何特定工具。 如果你使用其他支持 Skills（或类似机制）的 AI 编程助手，只需要：

  1. 确保系统已安装 Node.js（参见上方第 1 步）— 这是 Skills 依赖的 pagx-cli 工具运行所必需的。
  2. 将 Skills 文件拷贝到对应目录。参考上方第 3 步的解压过程，只需将 .codebuddy/skills/ 替换为你所用工具指定的 Skills 存放路径即可。

开始使用

Skills 安装完成后，你可以用自然语言与 AI 交互。以下是一些常见的操作示例：

创建 PAGX 文件

  • "创建一个蓝色圆角矩形按钮的 PAGX 文件，按钮上有白色文字 'Submit'"
  • "设计一个卡片组件，包含头像、标题和副标题"
  • "用简单的几何图形做一个设置图标，24x24 尺寸"
  • "创建一个渐变色横幅，上面有加粗文字 'Welcome Back'"

修改与迭代

  • "把按钮颜色改成绿色，圆角再大一些"
  • "标题文字调大一些，改成左对齐"
  • "给卡片加一个淡淡的阴影效果"
  • "图标和文字之间的间距感觉太紧了，拉开一些"

渲染与检查

  • "把这个 PAGX 文件渲染成截图，我看一下效果"
  • "渲染成 2 倍大小，方便在 Retina 屏幕上预览"
  • "校验一下这个 PAGX 文件有没有错误，有的话帮我修复"
  • "优化一下这个 PAGX 文件的体积"

在线预览

你也可以通过 PAGX Playground 在线预览 PAGX 文件 — 只需将任意 .pagx 文件拖放到页面上，即可在浏览器中查看渲染效果。

使用建议

多轮迭代，逐步趋近完美。 不要期望一次就得到完美结果，而是把这个过程当作一场对话：描述你想要的效果，预览渲染输出， 然后告诉 AI 哪里需要调整。每一轮反馈产生的都是精确修改 — 而不是概率性的重新抽卡 — 所以你会稳步逼近理想结果。这个多轮迭代过程可以完全交给 AI 自动完成， 也可以在任何环节让人介入，提供关键的审美判断和解构思路指导。 这种通过自然语言驱动、支持多轮迭代的精确修改能力，正是这套工作流程的核心价值所在。

以创作为主，而非临摹。 如果你提供位图作为参考，AI 会将其作为灵感来源进行全新的矢量创作，而不是逐像素描摹。 精确复现位图并非当前任何 AI 大模型的强项：位图本质上是一个基于概率的"感觉"， 而矢量图则是一个有唯一解的精确结构描述。从概率分布中求解唯一解， 从原理上与大语言模型的能力方向并不契合。 用参考图来传达风格和概念即可，让 AI 去创作一个干净的矢量演绎。
