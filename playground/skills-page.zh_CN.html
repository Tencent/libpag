简介

PAGX AI Skills 是面向 AI 编程助手的专业技能扩展包。安装后，AI Agent 可以根据文字描述生成结构简洁的 PAGX 矢量图形，优化现有文件的大小与渲染性能，并在需要时查询 PAGX 语法、属性和命令行工具用法。

当前主流的 AI 出图工具（如 Stable Diffusion、DALL-E）主要都是围绕像素生成位图，本质上是从概率分布中采样结果。若某个细节不理想，通常只能整张重试，很难只改动单一局部。

PAGX 是专门为 AI 优化的矢量图格式，AI 输出的是结构化标记而不是像素采样。文件中的元素都带有明确、可编辑的参数，因此你可以用自然语言指定局部改动，例如颜色、尺寸、位置和间距，并且不影响其他部分。这种“可精确二次编辑”的能力，是 AI 矢量工作流的核心优势。

这也决定了工作方式：围绕“目标描述 → 渲染验证 → 定点修改”进行多轮迭代。AI 负责高频执行，人工在关键轮次提供审美与结构方向，结果会逐步逼近上线质量。

PAGX 最初为动效描述而设计，动效能力建立在矢量静态图基础上。当前阶段重点仍是夯实静态能力；后续动效模块完善后，AI 可在同一套元素结构上继续生成更复杂的动画效果。

安装指南

PAGX Skills 目前主要在 CodeBuddy Code + Claude Opus 模型上完成测试。使用其他工具或模型时效果可能有所不同。

1. 安装 Node.js

Skills 安装包包含命令行工具 @libpag/pagx，运行依赖 Node.js。若本机尚未安装，请先完成以下步骤：

第 1 步 — 访问 nodejs.org，下载适用于你操作系统的 LTS（长期支持）版本。

第 2 步 — 运行安装程序，按照屏幕提示操作，保持所有默认设置即可。

第 3 步 — 验证安装是否成功。打开终端（macOS 上的"终端"应用，Windows 上的"命令提示符"或"PowerShell"），输入以下命令：

node --version

如果显示类似 v22.x.x 的版本号，说明 Node.js 已安装成功。

2. 安装 AI 编程助手

PAGX Skills 可用于任何支持 Skills 机制的 AI 编程助手。这里以 CodeBuddy Code 为例，其他工具请参考其官方文档完成安装。

第 1 步 — 打开终端，运行以下安装命令：

npm install -g @tencent-ai/codebuddy-code

第 2 步 — 安装完成后，输入以下命令启动：

codebuddy

按照屏幕提示完成初始设置和账号认证。

如需了解详细的安装说明和系统要求，请参阅 CodeBuddy Code 官方文档。

3. 安装 PAGX Skills

下载 pagx.zip 后，将其解压，把得到的 pagx 文件夹放到用户级 Skills 目录中。这样所有项目都可以使用该技能。

平台Skills 目录
macOS / Linux~/.codebuddy/skills/
WindowsC:\Users\<用户名>\.codebuddy\skills\

最终目录结构应如下所示：

.codebuddy/
  skills/
    pagx/
      SKILL.md
      references/
        ...

如果 .codebuddy/skills/ 文件夹尚不存在，手动创建即可。然后在任意项目目录下启动 CodeBuddy Code，PAGX Skills 会被自动识别并加载。

@libpag/pagx 命令行工具会在 AI 助手首次需要时自动安装，这依赖于第 1 步中搭建的 Node.js 环境。

开始使用

安装完成后，可直接用自然语言驱动 AI。下面按“创建→迭代→验证”给出常见示例：

创建 PAGX 文件

首次创建时，请在提示中包含 "PAGX" 关键词，以便 AI 激活 PAGX 技能。激活后，同一对话中的后续消息无需再重复该关键词。

  • "创建一个 PAGX 按钮：蓝色圆角矩形，白色文字 'Submit'"
  • "设计一个 PAGX 卡片，包含头像、标题和副标题"
  • "用简单几何图形做一个 PAGX 设置图标，24x24 尺寸"
  • "创建一个 PAGX 渐变色横幅，加粗文字 'Welcome Back'"

修改与迭代

  • "把按钮颜色改成绿色，圆角再大一些"
  • "标题文字调大一些，改成左对齐"
  • "给卡片加一个淡淡的阴影效果"
  • "图标和文字之间的间距感觉太紧了，拉开一些"

渲染与检查

  • "把这个 PAGX 文件渲染成截图，我看一下效果"
  • "渲染成 2 倍大小，方便在 Retina 屏幕上预览"
  • "校验一下这个 PAGX 文件有没有错误，有的话帮我修复"
  • "优化一下这个 PAGX 文件的体积"

使用建议

多轮迭代，稳定收敛到上线质量。这套流程的核心是：只要迭代次数足够多，结果理论上就能持续逼近上线标准。每一轮都围绕“明确目标 → 渲染验证 → 定点修改”推进；其中一部分可由 AI 自动完成，另一部分由人工在关键轮次提供审美与结构方向。随着大模型、Skills 和配套工具持续进步，人工参与的轮次会逐步减少，指令也会更简洁。

以创作为主，不以临摹为目标。当你提供位图参考时，建议把它用于传达风格、氛围和构图方向，而不是要求逐像素复刻。位图生成本质上是概率采样，输出更接近“感觉”；而矢量结果是参数明确、接近唯一解的结构表达。从概率分布中直接还原唯一结构，并不符合当前大模型的能力优势。实践中，更可靠的流程是先让 AI 产出结构正确的矢量版本，再通过局部参数持续打磨细节。
