## 二、第一轮遗留决策复查（6 项）

### P-1 Path 双格式（format=0 / format=1）

**第一轮建议**：砍 format=0（约 82 行，L4654-4735）  
**主决策者决定**：保留（溢出兜底）  

**本轮复查**：✅ **决策合理，无修改建议**

**复查理由**：
- format=0（裸 float）虽然比 format=1（量化）冗长，但有两个合理的实际用途：
  1. **短路径优化**（L4644-4645）：≤10 verb 的小 Path，format=1 的 5 byte 前缀开销相对过大，format=0 更优
  2. **坐标溢出兜底**（L4646-4647）：坐标绝对值 ≥ 2^23 导致量化后 int24 溢出时自动回退 format=0
- 代码中 `ChoosePathFormat()` 规则（L4642-4649）已明确这两个回退条件
- 双格式 + 自动选择策略是"最小必需"设计，而非"超前设计"

**注**：第一轮建议未充分量化这两个用途的实际收益（"小 Path 样本累积可省 10-15% body"见 L4652），建议者的论据可以更有力。
