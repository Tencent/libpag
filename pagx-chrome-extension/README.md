# PAGX Viewer - Chrome Extension

Preview PAGX vector graphics files directly in Chrome browser.

## Features

- Drag and drop `.pagx` files into the viewer page
- Drag `.pagx` files onto the Chrome Dock icon (macOS) or Taskbar icon (Windows) to open directly
- Split-view with PAGX source code viewer and syntax highlighting
- Draggable split divider with line numbers
- WebGL2 rendering via WASM engine

## Prerequisites

- Node.js (for building viewer.js)
- CMake (for building WASM module)
- Emscripten SDK (for WASM compilation)
- Google Chrome (for CRX packaging)

## Build

Run `build.sh` to build all components (WASM module, fonts, viewer bundle):

```bash
cd pagx-chrome-extension
bash build.sh
```

This script will:
1. Build the WASM module via the playground build system
2. Copy WASM artifacts (`wasm/`) from the build output
3. Copy font files (`fonts/`) from project resources
4. Install npm dependencies and compile `viewer.ts` into `viewer.js`

## Local Development

After building, load the extension in Chrome for development:

1. Open `chrome://extensions/`
2. Enable **Developer mode** (top right toggle)
3. Click **Load unpacked** and select the `pagx-chrome-extension/` directory
4. Click the extension details and enable **Allow access to file URLs**

> **Important**: The "Allow access to file URLs" permission is required for opening local `.pagx` files. Without it, dragging files to the Dock/Taskbar icon or opening `file://` URLs will not work.

### Opening PAGX Files

There are three ways to open a `.pagx` file:

1. **Drag & drop into viewer**: Open the extension's viewer page, then drag a `.pagx` file onto it
2. **Drag to Chrome icon**: Drag a `.pagx` file onto the Chrome icon in the macOS Dock or Windows Taskbar
3. **Open file URL**: Navigate to a `file:///path/to/file.pagx` URL in Chrome's address bar

Changes to `.css` and `.html` files take effect after refreshing the extension page.
Changes to `.ts` files require rebuilding:

```bash
npx rollup -c rollup.config.js
```

## Packaging

Use `package.sh` to create distributable packages. Build artifacts must exist before packaging (run `build.sh` first).

### CRX (local distribution)

CRX files can be shared with others for direct installation.

```bash
# Use an existing private key (required)
bash package.sh crx --key /path/to/pagx-viewer.pem

# Or use the default key location (pagx-viewer.pem in parent directory)
bash package.sh crx
```

The `--key` option specifies a `.pem` private key file. Using the same key across versions keeps the Extension ID consistent, allowing Chrome to recognize updates. If no key exists yet, generate one via Chrome:

1. Open `chrome://extensions/` and enable Developer mode
2. Click **Pack extension**, select the `pagx-chrome-extension/` directory
3. A `.pem` file will be generated alongside the `.crx`

Install a `.crx` file by dragging it into `chrome://extensions/`.

### ZIP (Chrome Web Store)

```bash
bash package.sh zip
```

Upload the generated `.zip` file to the [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole).

## Project Structure

```
pagx-chrome-extension/
├── manifest.json           # Extension manifest (Manifest V3)
├── background.js           # Service Worker
├── build.sh                # Build script
├── package.sh              # Packaging script (CRX / ZIP)
├── PRIVACY_POLICY.md       # Privacy policy for store submission
├── icons/                  # Extension icons (16, 48, 128)
├── viewer/
│   ├── viewer.html         # Viewer page structure
│   ├── viewer.css          # Styles (loaded directly, no build needed)
│   ├── viewer.ts           # Main viewer logic (source)
│   ├── viewer.js           # Compiled bundle (gitignored)
│   └── cache-manager.ts    # IndexedDB caching (source)
├── wasm/                   # WASM artifacts (gitignored, generated by build.sh)
├── fonts/                  # Font files (gitignored, copied by build.sh)
├── rollup.config.js        # Rollup bundler config
├── tsconfig.json           # TypeScript config
└── package.json            # npm config and dev dependencies
```
