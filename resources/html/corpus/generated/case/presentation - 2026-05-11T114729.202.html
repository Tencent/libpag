架构分析 / ARCHITECTURE
生产流水线逻辑 PAGE 06
SYSTEM ARCHITECTURE

从输入到 PPT 的六层架构

LAYER 01

输入层：接收用户材料

接收用户提供的主题、文本、图片、文件等多种格式的原始材料，作为系统处理的起点，确保各类信息源都能被统一接入和后续处理。

LAYER 02

理解层：读懂任务需求

深度分析用户输入，判断具体需求类型和任务目标，识别内容结构与关键意图，确保系统准确读懂任务要求，为后续生成路径选择提供依据。

LAYER 03

内容层：规划讲述逻辑

基于理解结果生成完整大纲，进行智能分页规划，提取每页核心观点与重点信息，决定整个演示文稿"讲什么"的内容框架与逻辑脉络。

LAYER 04

视觉层：统一呈现风格

统一选择设计风格、视觉语言体系和图表主题方案，建立色彩、字体、布局等视觉规范，决定内容"怎么讲"的视觉呈现方式与美学标准。

LAYER 05

页面层：生成可预览幻灯片

将内容与视觉方案转化为具体的 HTML 幻灯片代码，生成可实时预览的页面效果，确保每页布局合理、信息完整且视觉风格保持一致。

LAYER 06

导出层：输出最终文件

将生成的 HTML 幻灯片最终转换为标准 PPTX 文件格式，完成从网页预览到可编辑演示文稿的格式转换，支持下载或接入画布继续编辑。

GENERATION MODES

三大模式：topic、text、rich

Topic 模式：从零创作

Input Type

接收一个主题或一句话作为输入

Application

适用于主题分享、商业计划、课程内容等创作场景

System Capability

重点在于扩展内容并搭建框架，基于skills完成后台全流程自动编排，确保内容规划稳定、视觉风格统一，全流程使用公司内部部署模型。

Text 模式：文档转换

Input Type

处理长文本、报告、文章等已有文档材料

Application

适用于报告转PPT、文档转PPT等整理场景

System Capability

重点在于保留原文重点并实现稳定分页，确保长文档转换后的逻辑连贯性与版式一致性，支持从HTML预览到PPTX下载的跨平台链路。

Rich 模式：图文混排

Input Type

处理文本加图片的混合输入材料

Application

适用于产品介绍、截图讲解、图文材料等复杂场景

System Capability

重点实现图片理解与图文匹配，确保视觉元素与文字内容精准对应，提升演示材料的专业呈现效果与信息传达效率。

生成路径选择机制 PAGE 08
06

系统如何选择生成路径？

用户指令明确性

系统首先检测用户是否在输入中明确指定了生成方式。这种显式指令具有最高优先级，可直接决定生成路径，确保用户主观意图得到优先响应。

内容长度特征

系统分析输入文本的长度与结构特征，区分短主题输入（通常几句话或关键词）与长文本输入（大段段落、多章节内容），以匹配最优处理模式。

文档结构复杂度

识别文本中是否包含明确的标题层级、章节划分、表格数据或统计数字。检测到复杂结构时，将触发深度解析路径以维持逻辑严密性。

核心机制：非单因子决策 → 多维度综合权重判断

12

rich 模式：让图片真正服务内容

Phase 01

内容输入与多维解析

系统首先接收用户输入的文字描述和原始图片资源，随后对文本进行深度语义解析以提取关键主题，同时运用计算机视觉技术理解图片中的实体、场景与文字信息。

Phase 02

智能匹配与页面决策

基于解析结果，系统将图片内容与页面文字主题进行跨模态语义匹配，分析图文关联度与上下文契合度，进而智能决策每张图片最适合放置在哪一页、哪个具体位置。

Phase 03

页面生成与格式导出

系统生成带有精确图片占位符的HTML页面框架，定义图片尺寸、裁剪方式与环绕布局，随后将筛选后的真实图片注入对应位置并完成响应式适配。

DECISION LOGIC

什么时候需要图表？

必要性原则：避免为图表而图表
图表并非每页必需，核心目标是在数据真正适合可视化时，帮助页面更清楚地表达信息，避免形式主义。
适合生成图表的场景
数据分析意图
需要通过可视化呈现深层数据洞察，而非简单陈述。
多指标直观比较
存在多个数字、百分比，文字描述难以实现快速对比。
趋势与分布呈现
数据随时间变化或具有明显的分布特征。
不建议强行生成的类型
指南与攻略类
普通说明页、旅行攻略、景点介绍、美食推荐等。
缺乏数据支撑
仅为装饰而设，或没有可靠量化数据支撑的页面。
纯文本叙事
强调情感表达或逻辑推演，而非量化分析的场景。

VISUAL DESIGN SYSTEM PAGE 15

Visual Prompt：整套 PPT 的

Core Definition

核心定义

Visual Prompt 是给 AI 的视觉设计规范，它决定整套 PPT 的视觉语言。

色彩系统规范

严格规定主色、辅助色和背景色的使用规则与搭配比例，确保色彩应用符合品牌识别或场景需求。通过标准化的色值定义，杜绝随机配色导致的视觉混乱。

元素气质定义

规范图表类型、图标风格及装饰元素的设计气质与细节处理，使所有视觉组件在圆角、描边、阴影等维度保持高度一致，确保整体视觉语言连贯专业。

布局逻辑控制

定义页面的呼吸感、留白节奏与信息层级分布，确保不同内容页在视觉重心上保持统一的逻辑引导。

视觉资产标准

统一图像处理风格、材质感与光影方向，使所有外部引入的视觉素材在进入 PPT 后能迅速同化为整体风格的一部分。
